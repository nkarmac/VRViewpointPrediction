Reading Summary:
Rotational Motion Model for
Temporal Prediction in 360 Video Coding

Nicolaus Karamanian, CSC 499
March 27th, 2020

The Problem

  Due to the enormous amount of data consumed by spherical video, it is imperative to efficiently compress video data for storage and transmission. Current approaches simply project the spherical video onto 2D planes to be processed by 2D video coding standards. This is significantly suboptimal due to the skewed frames from the translation. Translation in the projected domain does not preserve an object’s shape and size, meaning that motion vectors are not useful predictions.

Main Idea

  Take advantage of the fact that the video is a sphere. Characterize or predict the motion directly on the sphere with a rotational motion model (as opposed to the regular translational motion model). Motion vectors specify axis and degree of rotation, where motion search is a radial search pattern. Rotation of the frame with respect to the predicted motion vectors is done along the geodesic via the Rodrigues’ rotation formula (an efficient method for 3D vector rotation). The rotated frame is then mapped/projected back to the reference frame in projected geometry, where performing interpolation obtains the required prediction.

Major Strengths

  • The method averages a higher bitrate savings in all tested cases in comparison to currently used 2D coding standards (such as HEVC).
  • Motion prediction directly on the sphere removes the problem of false “warped” objects based on post-projection prediction.
  • The implementation, while being suggested to replace 2D prediction, can be used in addition for additional accuracy.

Major Weaknesses

  • In this study, they use only the previous frame as a reference frame. Many 2D coding standards, that they are comparing against, utilize multiple previous frames for a motion prediction. It is whether the standards they compared against predicted based on a single frame or not.
  • It is unclear why calculating bitrate savings is the best way to compare this method relative to currently used 2D models.
  • It is also unclear how the proposed motion search model could be implemented in a working system.

Possible Improvements

  Only the previous frame is used as a reference frame in their experiment, perhaps expanding this method to interpolate based on multiple frames will improve results.
