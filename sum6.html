Reading Summary:
Motion-Prediction-Based Multicast
for 360-Degree Video Transmissions (Viewpoint Prediction Portion)

Nicolaus Karamanian, CSC 499
March 25th, 2020

The Problem

  360-degree videos have a size 4-5x that of regular 2D videos. Each viewer has to wear a Head-Mounted Display (HMD), and since screen sharing is impossible, a group watching the same content requires multiple parallel video-streams. Due to the large increase in bandwidth for even one video, multiple parallel streams would choke an entire network.

Main Idea

  A motion-prediction based multicast serves concurrent viewers. Viewers typically show similar motion patterns when watching the same 360 videos, so perform temporal prediction of view motion based on previous states to optimize bandwidth. Obtain X,Y, and Z angles from individual HMDs, and record each user’s previous angles and motions. Train a neural network for a future user angle/motion prediction model that over-samples cases with large errors. Generate a predicted FOV for each individual viewer with an accuracy of at least 99.9% (increase predicted FOV size until accuracy met). Use a “smart” multicast to unicast the predicted frame portion to each individual viewer, and if multiple users have the same FOV, multicast that portion (unless individual unicasts cost less).

Major Strengths

  • Great bandwidth reduction results of over 50% when compared to full multicasting.
  • The 99.9% accuracy requirement assures that the same full quality before the method is applied is always maintained.
  • Due to switching between multicasting and unicasting based on cost, the room for error is large (a large error falls back to the base case of full multicasting).

Major Weaknesses

  • This method computes video compression online, especially when switching to multiple unicast streaming, which greatly raises computation and may cause more unexpected stutters than typical video loading.
  • Multicasting has no retransmission, and because this method uses previous online data to predict future data, such predictions can be completely wrong if the previous frames were not transmitted/corrupt.
  • They focus on correcting large errors by not considering Mean Square Error for the regression model, stating that large error corrections could cover small errors. The accuracy requirement reduces this problem, but it is unclear whether many small errors is a better trade-off for few large ones in this case.

Possible Improvements

  If possible, use storage to trade for online computation, perhaps by preprocessing the different viewpoint’s compressed frames offline (as opposed to online like proposed).
